# 配置表编辑器使用指南

## 概述

配置表编辑器是一个可视化工具，用于：
1. 从现有 Excel 文件加载配置表结构
2. 定义配置表字段并生成模板说明
3. 自动生成对应的 C# 代码

**注意**：此工具不直接创建/修改 Excel 文件，需要使用 Microsoft Excel、WPS 或其他 Excel 工具来创建实际的 Excel 文件。

## 打开编辑器

在 Unity 菜单栏选择：`Tools > Excel > 配置表编辑器`

## Excel 格式规范

### 标准格式（4行）

```
行0: 注释行（字段说明）
行1: 字段名行
行2: 类型定义行
行3+: 数据行
```

### 示例

| 物品ID | 名称 | 类型 | 品质 | 图标 |
|--------|------|------|------|------|
| Id | Name | Type | Quality | Icon |
| int | string | int | int | string |
| 1001 | 铁剑 | 1 | 2 | icon_sword |
| 1002 | 木盾 | 2 | 1 | icon_shield |

## 工作模式

### 模式一：LoadAndGenerate（加载并生成）

从现有 Excel 文件加载表结构并生成代码。

**适用场景**：
- 已有 Excel 文件，需要生成代码
- 修改了 Excel 结构，需要重新生成代码

### 模式二：CreateTemplate（创建模板）

定义配置表结构，生成代码和模板说明文件。

**适用场景**：
- 创建新的配置表
- 需要模板说明来指导 Excel 创建

## 功能一：从现有 Excel 生成代码

### 步骤

1. **选择操作模式**
   - 在"操作模式"下拉框中选择"LoadAndGenerate"

2. **设置 Excel 路径**
   - 输入 Excel 文件路径，或点击"浏览"按钮选择
   - 例如：`Assets/RefData_Excel/ItemConfig.xlsx`

3. **输入表名**（可选）
   - 如果不输入，会使用第一个工作表
   - 如果输入，会查找指定名称的工作表

4. **加载表结构**
   - 点击"加载 Excel 表结构"按钮
   - 系统会读取 Excel 并显示字段定义

5. **检查字段定义**
   - 确认字段名、类型、注释是否正确
   - 可以调整字段顺序或修改定义

6. **生成代码**
   - 点击"生成代码"按钮
   - 系统会生成 C# 配置类代码

### 示例

```
Excel 路径: Assets/RefData_Excel/ItemConfig.xlsx
表名: ItemConfig

加载后显示:
[✓] Id        int      物品ID
[ ] Name      string   名称
[ ] Type      int      类型
[ ] Quality   int      品质
[ ] Icon      string   图标路径
```

## 功能二：创建新配置表

### 步骤

1. **选择操作模式**
   - 在"操作模式"下拉框中选择"CreateTemplate"

2. **设置 Excel 路径**
   - 输入将要创建的 Excel 文件路径
   - 例如：`Assets/RefData_Excel/NewConfig.xlsx`

3. **输入表名**
   - 输入配置表名称
   - 例如：`SkillConfig`

4. **定义字段**
   - 点击"+ 添加字段"按钮添加字段
   - 配置每个字段：
     - **主键**：勾选复选框标记为主键
     - **字段名**：输入字段名称
     - **类型**：从下拉框选择类型
     - **注释**：输入字段说明

5. **调整字段顺序**
   - 使用"↑"和"↓"按钮调整顺序
   - 使用"删除"按钮移除字段

6. **导出模板说明**
   - 点击"导出模板说明"按钮
   - 系统会生成一个 `.txt` 文件，包含详细的创建说明

7. **按照模板创建 Excel**
   - 使用 Microsoft Excel 或 WPS 打开/创建 Excel 文件
   - 按照模板说明填写前3行
   - 从第4行开始填写数据
   - 保存文件

8. **生成代码**
   - 返回 Unity
   - 切换到"LoadAndGenerate"模式
   - 加载刚创建的 Excel 文件
   - 生成代码

### 模板说明文件示例

```
========================================
配置表模板说明: SkillConfig
========================================

Excel 文件路径: Assets/RefData_Excel/SkillConfig.xlsx
工作表名称: SkillConfig

========================================
Excel 格式规范
========================================

行0: 注释行（字段说明）
行1: 字段名行
行2: 类型定义行
行3+: 数据行

========================================
字段定义
========================================

注释行（第1行）:
| 技能ID          | 技能名称        | 伤害范围        |

字段名行（第2行）:
| Id              | Name            | Damage          |

类型定义行（第3行）:
| int             | string          | IntRange        |

数据示例行（第4行）:
| 1001            | 火球术          | 100-200         |

========================================
创建步骤
========================================

1. 使用 Microsoft Excel 或 WPS 打开/创建 Excel 文件
2. 创建名为 'SkillConfig' 的工作表
3. 按照上面的格式填写前3行（注释、字段名、类型）
4. 从第4行开始填写数据
5. 保存文件
6. 返回 Unity，使用配置表编辑器加载并生成代码
```

### 支持的类型

#### 基本类型

| 类型 | 说明 | 示例 |
|------|------|------|
| int | 整数 | 100 |
| long | 长整数 | 1000000 |
| float | 单精度浮点数 | 3.14 |
| double | 双精度浮点数 | 3.14159 |
| string | 字符串 | "Hello" |
| bool | 布尔值 | true/false |

#### 数组类型

| 类型 | 说明 | 示例 |
|------|------|------|
| int[] | 整数数组 | 1,2,3 |
| float[] | 浮点数数组 | 1.0,2.0,3.0 |
| string[] | 字符串数组 | a,b,c |

#### Unity 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| Vector2 | 2D向量 | 100,200 |
| Vector3 | 3D向量 | 100,200,300 |
| Color | 颜色 | #FF0000 或 255,0,0 |

#### 自定义类型

| 类型 | 说明 | 示例 |
|------|------|------|
| ItemReward | 物品奖励 | 1001:10 |
| RewardList | 奖励列表 | 1001:10;1002:5 |
| IntRange | 整数范围 | 100-200 |
| ItemConfig[] | 自定义类型数组 | 支持任何自定义类型的数组 |

**类型输入方式**：
- **文本输入框**：直接输入任何类型名称（支持自定义类型）
- **快速按钮**：
  - `int` - 设置为 int 类型
  - `float` - 设置为 float 类型
  - `string` - 设置为 string 类型
  - `[]` - 在当前类型后添加数组后缀

**示例**：
- 输入 `ItemConfig` 表示自定义类型
- 输入 `ItemConfig[]` 表示自定义类型数组
- 点击 `[]` 按钮可以快速添加数组后缀

详细的类型定义说明请参考 [类型定义说明.md](类型定义说明.md)

## 生成的代码

### 配置类

```csharp
[Table("ItemConfig")]
[Serializable]
public class ItemConfig
{
    [PrimaryKey]
    [Column("Id")]
    public int Id { get; set; }
    
    [Column("Name")]
    public string Name { get; set; }
    
    [Column("Type")]
    public int Type { get; set; }
    
    [Column("Quality")]
    public int Quality { get; set; }
    
    [Column("Icon")]
    public string Icon { get; set; }
}
```

### 加载类

```csharp
public class ItemConfigTable : ConfigBase<int, ItemConfig>
{
    public ItemConfigTable()
    {
        // 可以在这里指定数据库路径和表名
        // Load(dbPath, "ItemConfig");
    }
    
    protected override int GetKey(ItemConfig item)
    {
        return item.Id;
    }
}
```

   ↓

## 最佳实践

### 1. 命名规范

- **表名**：使用 PascalCase，如 `ItemConfig`, `SkillConfig`
- **字段名**：使用 PascalCase，如 `Id`, `Name`, `ItemType`
- **注释**：使用中文，简洁明了

### 2. 类型选择

- **ID 字段**：使用 `int` 或 `long`
- **名称描述**：使用 `string`
- **数值**：根据范围选择 `int`, `long`, `float`, `double`
- **开关标记**：使用 `bool`
- **复杂数据**：使用自定义类型

### 3. 主键设置

- 每个表必须有至少一个主键
- 通常第一个字段（Id）作为主键
- 主键值必须唯一

### 4. 字段顺序

- 主键字段放在最前面
- 重要字段靠前
- 相关字段放在一起

### 5. 注释规范

- 每个字段都应该有注释
- 注释要简洁明了
- 对于枚举类型，注释中说明取值范围

## 常见问题

### Q1: 如何添加自定义类型？

A: 现在支持直接在类型输入框中输入任何类型名称：
1. 在类型输入框中直接输入自定义类型名，如 `ItemConfig`, `RewardData`
2. 如果是数组类型，添加 `[]` 后缀，如 `ItemConfig[]`
3. 可以使用快速按钮 `[]` 自动添加数组后缀
4. 确保自定义类型实现了 `ICustomTypeParser` 接口或有静态 `Parse` 方法

详细说明请参考 [类型定义说明.md](类型定义说明.md)

### Q2: 修改字段定义后数据会丢失吗？

A: 不会。修改只影响前3行（注释、字段名、类型），数据行不受影响。

### Q3: 如何删除字段？

A: 在编辑器中点击"删除"按钮，然后保存修改。Excel 中的数据列仍会保留。

### Q4: 可以修改主键吗？

A: 可以。取消勾选当前主键，勾选新的主键字段即可。

### Q5: 生成的代码在哪里？

A: 默认保存在 `Assets/Scripts/HotUpdate/Config/` 目录下。

### Q6: 如何使用生成的配置类？

A: 
```csharp
// 创建加载器
var itemTable = new ItemConfigTable();
itemTable.Load(dbPath, "ItemConfig");

// 查询数据
var item = itemTable.GetByKey(1001);
Debug.Log($"物品名称: {item.Name}");
```

## 注意事项

1. **ExcelDataReader 依赖**：编辑器使用 ExcelDataReader 库读取 Excel，需要先安装
2. **不支持直接创建 Excel**：此工具不能直接创建/修改 Excel 文件，需要使用外部工具（Microsoft Excel、WPS 等）
3. **代码覆盖**：生成代码会覆盖同名文件，请注意备份
4. **类型安全**：确保 Excel 中的数据类型与定义的类型匹配
5. **主键唯一**：确保主键值在数据中唯一
6. **Excel 格式**：必须严格按照4行格式（注释、字段名、类型、数据）创建 Excel
