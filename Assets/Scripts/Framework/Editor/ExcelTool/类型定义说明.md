# 配置表类型定义说明

## 概述

配置表编辑器支持灵活的类型定义方式，可以使用基础类型、数组类型、Unity 类型和自定义类型。

## 类型输入方式

### 1. 文本输入框
类型字段现在是一个**文本输入框**，你可以直接输入任何类型名称：
- 基础类型：`int`, `float`, `string`, `bool`, `long`, `double`
- 数组类型：`int[]`, `float[]`, `string[]`
- Unity 类型：`Vector2`, `Vector3`, `Color`
- 自定义类型：`ItemConfig`, `RewardData`, `SkillInfo` 等
- 自定义数组：`ItemConfig[]`, `RewardData[]` 等

### 2. 快速选择按钮
每个类型输入框下方有快速选择按钮：
- **int** - 设置为 int 类型
- **float** - 设置为 float 类型
- **string** - 设置为 string 类型
- **[]** - 在当前类型后添加数组后缀 `[]`

## 支持的类型

### 基础类型
| 类型 | 说明 | Excel 示例 |
|------|------|-----------|
| int | 整数 | 1001 |
| long | 长整数 | 9999999999 |
| float | 单精度浮点数 | 1.5 |
| double | 双精度浮点数 | 3.14159 |
| string | 字符串 | 示例文本 |
| bool | 布尔值 | true / false / 1 / 0 |

### 数组/列表类型
| 类型 | 说明 | Excel 示例 |
|------|------|-----------|
| int[] | 整数数组 | 1,2,3,4,5 |
| float[] | 浮点数数组 | 1.0,2.5,3.7 |
| string[] | 字符串数组 | a,b,c |
| List<int> | 整数列表（推荐） | 1,2,3,4,5 |
| List<float> | 浮点数列表（推荐） | 1.0,2.5,3.7 |
| List<string> | 字符串列表（推荐） | a,b,c |

**注意**：
- 数组元素使用**逗号**分隔
- List 和 Array 的 Excel 格式完全相同
- **推荐使用 List**，提供更好的 API

### Unity 内置类型
| 类型 | 说明 | Excel 示例 |
|------|------|-----------|
| Vector2 | 二维向量 | 100,200 |
| Vector3 | 三维向量 | 100,200,300 |
| Color | 颜色 | #FF0000 或 255,0,0,255 |

### 自定义类型

#### 使用 ICustomTypeParser
如果你的自定义类型实现了 `ICustomTypeParser` 接口或有 `Parse` 方法，可以直接使用：

```csharp
// 示例：ItemReward 类型
[CustomTypeParser(typeof(ItemRewardParser))]
public class ItemReward
{
    public int ItemId;
    public int Count;
}

// Excel 中的格式：1001:10
```

#### 常用自定义类型示例
| 类型 | 说明 | Excel 示例 |
|------|------|-----------|
| ItemReward | 物品奖励 | 1001:10 |
| RewardList | 奖励列表 | 1001:10;1002:5 |
| IntRange | 整数范围 | 100-200 |
| KeyValuePair | 键值对 | key:value |

#### 自定义类型数组/列表
自定义类型也支持数组和列表：

| 类型 | 说明 | Excel 示例 |
|------|------|-----------|
| ItemReward[] | 物品奖励数组 | 1001:10;1002:5;1003:3 |
| List<ItemReward> | 物品奖励列表（推荐） | 1001:10;1002:5;1003:3 |
| Vector3[] | 向量数组 | 0,0,0;100,0,0;100,100,0 |
| List<Vector3> | 向量列表（推荐） | 0,0,0;100,0,0;100,100,0 |

**注意**：
- 数组/列表元素使用**分号**分隔
- 每个元素内部使用**逗号**或**冒号**分隔
- **推荐使用 List**，提供 Add、Remove、Find 等便利方法

## 类型定义最佳实践

### 1. 命名规范
- 基础类型使用小写：`int`, `float`, `string`
- 自定义类型使用 PascalCase：`ItemConfig`, `RewardData`
- 数组类型添加 `[]` 后缀：`int[]`, `ItemConfig[]`

### 2. 类型选择建议
- **简单数值**：使用 `int` 或 `float`
- **文本内容**：使用 `string`
- **开关标记**：使用 `bool`
- **列表数据**：使用数组类型 `int[]`, `string[]`
- **复杂结构**：使用自定义类型 `ItemReward`, `SkillData`

### 3. 自定义类型使用流程
1. 在代码中定义自定义类型类
2. 实现 `ICustomTypeParser` 接口或静态 `Parse` 方法
3. 在配置表编辑器中直接输入类型名
4. 在 Excel 中按照解析器定义的格式填写数据

## 分隔符规范

| 分隔符 | 用途 | 示例 |
|--------|------|------|
| 逗号 (,) | 同级元素分隔 | 坐标、颜色、数组 |
| 冒号 (:) | 键值对 | ItemId:Count |
| 分号 (;) | 记录/组分隔 | 多个奖励、多个颜色 |
| 竖线 (\|) | 复杂结构字段 | 多层嵌套数据 |
| 星号 (*) | 权重 | 权重随机 |
| 减号/波浪 (-/~) | 范围 | 数值范围 |

## 常见问题

### Q: 如何定义数组/列表类型？
A: 
- **数组**：在类型名后添加 `[]`，例如 `int[]`, `ItemConfig[]`
- **列表**（推荐）：使用 `List<类型>`，例如 `List<int>`, `List<ItemConfig>`
- 可以使用快速按钮 **[]** 添加数组后缀，然后手动改为 List

**推荐使用 List**，因为它提供更多便利方法（Add、Remove、Find 等）。

### Q: 自定义类型找不到怎么办？
A: 确保：
1. 自定义类型类已经定义
2. 实现了 `ICustomTypeParser` 接口或有静态 `Parse` 方法
3. 类型名拼写正确（区分大小写）

### Q: 如何知道自定义类型的 Excel 格式？
A: 查看自定义类型的 `Parse` 方法实现，或参考 `CustomTypeParser_Guide.md` 文档。

### Q: 可以嵌套数组吗？
A: 不建议使用嵌套数组（如 `int[][]`），建议使用自定义类型来表示复杂结构。
