# Excel 配置表分隔符使用规范

## 📋 分隔符定义

| 分隔符 | 用途 | 示例 | 说明 |
|--------|------|------|------|
| `,` 逗号 | 同级元素分隔 | `100,200,300` | 用于数组、坐标、颜色等 |
| `:` 冒号 | 键值对 | `1001:10` | 表示 itemId:count |
| `;` 分号 | 记录/组分隔 | `1001:10;1002:5` | 分隔不同的记录或组 |
| `\|` 竖线 | 复杂结构字段 | `Damage\|100\|Fire` | 分隔复杂对象的字段 |
| `*` 星号 | 权重表示 | `item1*10` | 表示 item*weight |
| `-` 或 `~` | 范围 | `100-200` | 表示数值范围 |

## 🎯 使用场景

### 1. 坐标和向量（使用逗号）

```
| 位置      | 旋转        | 缩放      |
|----------|------------|----------|
| 100,200  | 0,90,0     | 1,1,1    |
| 150,250  | 0,180,0    | 2,2,2    |
```

### 2. 颜色（使用逗号）

```
| 背景颜色      | 文字颜色    |
|-------------|-----------|
| #FF0000     | 255,255,255 |
| 255,0,0     | 1.0,1.0,1.0 |
| 255,0,0,128 | #FFFFFF   |
```

### 3. 键值对（使用冒号）

```
| 消耗      | 奖励      |
|----------|----------|
| 1001:10  | 2001:5   |
| 1002:20  | 2002:10  |
```

### 4. 多个记录（使用分号）

```
| 多个奖励              | 多个消耗              |
|---------------------|---------------------|
| 1001:10;1002:5      | 2001:1;2002:2       |
| 1003:1;1004:2;1005:3| 2003:5              |
```

### 5. 复杂结构（使用竖线）

```
| 技能效果                    |
|---------------------------|
| Damage\|100\|Fire         |
| Heal\|50                  |
| Buff\|Speed\|10           |
```

### 6. 权重配置（使用星号和分号）

```
| 掉落物品                  |
|-------------------------|
| gold*50;silver*30;bronze*20 |
| sword*10;shield*5       |
```

### 7. 数值范围（使用减号或波浪）

```
| 伤害范围  | 金币奖励  | 等级要求  |
|----------|----------|----------|
| 100-200  | 50-100   | 10-20    |
| 300-500  | 100~200  | 20~30    |
```

## 🔄 组合使用

### 示例 1：多个奖励，每个奖励有ID和数量

```
格式：itemId:count;itemId:count;itemId:count
示例：1001:10;1002:5;1003:1
```

### 示例 2：多个技能效果

```
格式：type|param1|param2;type|param1|param2
示例：Damage|100|Fire;Buff|Speed|10
```

### 示例 3：权重掉落列表

```
格式：item*weight;item*weight;item*weight
示例：gold*50;silver*30;bronze*20
```

### 示例 4：多个坐标点

```
格式：x,y;x,y;x,y
示例：100,200;150,250;200,300
```

## ⚠️ 注意事项

### 1. 避免分隔符冲突

❌ **错误示例**：
```
// 在坐标中使用分号会导致解析错误
100;200;300  // 应该使用逗号：100,200,300
```

✅ **正确示例**：
```
// 坐标使用逗号，多个坐标使用分号
100,200;150,250;200,300
```

### 2. 保持一致性

在同一个配置表中，相同类型的数据应该使用相同的分隔符。

❌ **不一致**：
```
| 奖励1        | 奖励2        |
|-------------|-------------|
| 1001:10     | 1002,5      |  // 混用冒号和逗号
```

✅ **一致**：
```
| 奖励1        | 奖励2        |
|-------------|-------------|
| 1001:10     | 1002:5      |  // 统一使用冒号
```

### 3. 嵌套层级

建议最多 2-3 层嵌套，过深的嵌套会降低可读性。

✅ **推荐**（2层）：
```
1001:10;1002:5;1003:1
```

✅ **可接受**（3层）：
```
Damage|100|Fire;Buff|Speed|10
```

❌ **不推荐**（4层以上）：
```
quest|1|reward|1001:10,1002:5|condition|level:10,gold:100
// 太复杂，建议拆分成多个字段或使用 JSON
```

## 📝 快速参考表

### 常用数据类型格式

| 数据类型 | 格式 | 示例 |
|---------|------|------|
| 整数 | `123` | `100` |
| 浮点数 | `123.45` | `3.14` |
| 字符串 | `text` | `Hello` |
| 布尔值 | `true/false` 或 `1/0` | `true` |
| 数组 | `v1,v2,v3` | `1,2,3` |
| 键值对 | `key:value` | `1001:10` |
| 范围 | `min-max` | `100-200` |
| 坐标2D | `x,y` | `100,200` |
| 坐标3D | `x,y,z` | `100,200,300` |
| 颜色 | `r,g,b` 或 `#RRGGBB` | `255,0,0` 或 `#FF0000` |
| 权重 | `item*weight` | `gold*50` |
| 多记录 | `record1;record2` | `1001:10;1002:5` |
| 复杂结构 | `field1\|field2\|field3` | `Damage\|100\|Fire` |

## 🎓 最佳实践

### 1. 选择合适的分隔符

- **简单列表**：使用逗号 `,`
- **键值对**：使用冒号 `:`
- **多个记录**：使用分号 `;`
- **复杂结构**：使用竖线 `|`

### 2. 保持简洁

优先使用简单的分隔符组合，避免过度复杂的嵌套。

### 3. 添加注释

在 Excel 的注释行（第一行）中说明格式：

```
| 奖励列表 (格式: itemId:count;itemId:count) |
|-----------------------------------------|
| 1001:10;1002:5                          |
```

### 4. 提供示例

在配置表的第一行数据中提供标准示例，方便策划参考。

### 5. 错误提示

在 Parse 方法中提供清晰的错误提示：

```csharp
if (parts.Length != 2)
{
    throw new FormatException(
        $"格式错误: '{value}'\n" +
        $"期望格式: itemId:count\n" +
        $"示例: 1001:10"
    );
}
```

## 🔧 扩展建议

如果需要更复杂的配置，考虑：

1. **拆分字段**：将复杂数据拆分成多个简单字段
2. **使用 JSON**：对于非常复杂的嵌套结构，使用 JSON 格式
3. **引用其他表**：使用外键引用其他配置表
4. **自定义解析器**：为特殊格式实现专门的解析器
