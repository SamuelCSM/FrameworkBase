# 网络层外部环境验收清单（真机 / 真实网络）

> 环回 TCP/TLS 集成测试（`NetworkLoopbackTests`）覆盖真实 Socket/TLS 生命周期，但以下场景**无法用普通
> CI 或环回测试等价复现**，必须在真机 / 真实网络下人工验收。**不得以单元测试通过替代真机结论。**
> 每次涉及网络栈、TLS 证书、连接策略的改动发布前，至少抽样执行本清单相关项。

## 一、验收前置

- 构建：Development Build（保留日志）+ Release Build 各一份（正式包封死 PlayerPrefs 覆盖更新源，须分别验）。
- 抓包：设备侧 tcpdump / Charles / Wireshark，确认实际协议栈行为与日志一致。
- 观察点：连接建立耗时、重连次数与退避间隔、`ConnectionEpoch` 递增、请求超时与取消、内存（ArrayPool 无泄漏）。

## 二、必测外部场景

| 编号 | 场景 | 期望行为 | 通过判据 |
| --- | --- | --- | --- |
| N1 | NAT64-only 网络（iOS 审核要求） | IPv6 优先，DNS64/NAT64 下正常连通 | 无 IPv4 字面量假设；连接成功 |
| N2 | IPv6-only 网络 | 仅 AAAA 记录时正常连通 | 连接成功，日志 addressFamily=InterNetworkV6 |
| N3 | 双栈部分地址不可达 | 某地址失败后回退其余地址 | 单地址失败不致整体失败 |
| N4 | Wi-Fi ↔ 蜂窝切换 | 旧连接断开、新连接重建、旧 Epoch 数据被丢弃 | Epoch 递增；无错乱响应 |
| N5 | Android 前后台切换 | 回后台暂停/回前台重连并重鉴权 | 回前台后请求恢复；无重复补发非幂等请求 |
| N6 | iOS 前后台切换 | 同 N5（含 iOS 挂起 socket 语义） | 同 N5 |
| N7 | 弱网高延迟（RTT 秒级） | 请求超时与心跳半开检测按配置生效 | 超时回调触发；无误判断线 |
| N8 | 丢包 | 半开检测发现死连接并重连 | 死连接被识别；重连成功 |
| N9 | 网络瞬断（拔网/飞行模式抖动） | 指数退避 + 抖动重连；预算耗尽后停止并可手动重试 | 退避间隔符合配置；预算耗尽有明确终止 |
| N10 | DNS 污染 / 多地址部分不可达 | 遍历解析地址，跳过不可达 | 最终连通或明确失败，不永久挂起 |
| N11 | TLS 真实证书轮换（生产 CA） | 双 Pin 窗口内新旧证书都通过；切换后旧 Pin 客户端按预期被拒并提示更新 | 轮换零断连；旧客户端有明确失败提示 |
| N12 | 真实 CA 主机名不匹配 | 系统信任下主机名校验失败即拒绝（环回自签名无法隔离该项） | 连接被拒 |

## 三、与自动化测试的对应关系（避免重复投入）

- 已由 `NetworkLoopbackTests` 覆盖（真实 Socket/TLS，CI 内执行）：连接成功 / 服务端断开 / 断线重连 /
  世代隔离 / 半包 / 粘包 / 非法帧关闭 / 超大帧拒绝 / 背压失败 / 连接取消 / 连接超时 / TLS 系统信任失败 /
  Pin 匹配放行 / Pin 命中不放行系统信任仍拒 / Pin 不匹配 / 双 Pin 命中 / 证书轮换（双 Pin→移除旧 Pin）/
  TLS 握手超时。
- 已由 `NetworkHardeningTests` / `OfflineRequestQueueTests` 覆盖：入站队列上限、每帧预算、Epoch+SeqId
  匹配、请求取消、SeqId 回绕、IPv6 优先排序、离线队列仅补发显式 opt-in、非幂等请求不默认补发。
- 仅本清单（真机）：N1–N12。重连预算耗尽后的退避收敛与前后台重鉴权失败退避的**真机时序**属此列。
